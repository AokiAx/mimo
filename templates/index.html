认证

数据面写请求（/v1/chat/completions、/v1/responses、/v1/messages）需要：

Authorization: Bearer sk-Aoki-MiMo

网关会自动注入后端 API key，客户端只需提供网关 token。/v1/models 与 /health 公开可读；/api/public/status 使用独立的 status_api_token。

格式转换

OpenAI Chat / Responses：走统一适配链路，/v1/responses 会转换到上游 Chat 格式并路由到最佳可用后端。

Anthropic Messages：/v1/messages 先做模型映射和能力校验，再直连上游 Anthropic 原生接口，保留 thinking、signature 与 beta header。

🔑 密钥管理

在此查看 / 修改 / 轮换面板与接口密钥（持久化到 data/secrets.json，改完即时生效，无需重启）。
